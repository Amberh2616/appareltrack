## 1. 系統概述

### 1.1 系統定位

**AI-Augmented PLM + ERP Lite for Garment Factories**

本系統專為成衣廠設計，整合 AI 技術實現：
- Tech Pack 自動解析與翻譯
- BOM 物料表智能提取
- 尺寸規格自動識別
- 樣衣生產流程管理
- 大貨訂單與採購管理

**目標：** 1 人管理 300-500+ 款/季，70-80% 自動化

### 1.2 服務地址

| 服務 | URL |
|------|-----|
| 前端系統 | http://localhost:3000 |
| 後端 API | http://localhost:8000/api/v2/ |
| 管理後台 | http://localhost:8000/admin/ |

### 1.3 瀏覽器支援

- Google Chrome（推薦）
- Microsoft Edge
- Firefox
- Safari

## 4. Upload 上傳文件

**路徑：** `/dashboard/upload`

### 4.1 功能說明

上傳 Tech Pack、BOM 等文件，支援單筆和批量上傳。

### 4.2 介面佈局

```
┌─────────────────────────────────────────────────────────────┐
│ Upload Documents                                            │
├─────────────────────────────────────────────────────────────┤
│ [📄 單筆上傳]  [📦 批量上傳]                    ← Tab 切換  │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   ┌─────────────────────────────────────────────────────┐   │
│   │                                                     │   │
│   │        📄 拖拽文件到此處，或點擊選擇文件            │   │
│   │                                                     │   │
│   │           支援 PDF 格式，最大 50MB                  │   │
│   │                                                     │   │
│   └─────────────────────────────────────────────────────┘   │
│                                                             │
│   文件名稱：[________________________]                      │
│   文件類型：[Tech Pack ▼]                                   │
│   品牌：    [Lululemon ▼]                                   │
│                                                             │
│                              [上傳並處理]                   │
└─────────────────────────────────────────────────────────────┘
```

### 4.3 操作步驟

#### 單筆上傳
1. 確認在「單筆上傳」Tab
2. 拖拽 PDF 文件到虛線框，或點擊選擇文件
3. 填寫文件名稱（可選，預設使用檔名）
4. 選擇文件類型：Tech Pack / BOM / Mixed
5. 選擇品牌（可選）
6. 點擊「上傳並處理」
7. 等待上傳進度條完成
8. 自動跳轉至 AI 處理頁面

#### 批量上傳
1. 切換到「批量上傳」Tab
2. 準備 ZIP 壓縮檔（包含多個 PDF）
3. 拖拽或選擇 ZIP 檔案
4. 點擊「上傳並處理」
5. 系統自動解壓並逐一處理

### 4.4 支援格式

| 類型 | 格式 | 大小限制 |
|------|------|----------|
| 單筆 | PDF | 50 MB |
| 批量 | ZIP（內含 PDF）| 200 MB |

| 操作 | 說明 |
|------|------|
| 點擊 Tab | 切換不同分類的文件或款式 |
| 點擊「👁」| 查看文件或款式詳情 |
| 點擊「🗑」| 刪除重複或錯誤的文件 |

### 5.6 款式詳情頁

點擊款式的「👁」進入詳情後可查看：
- 基本資訊（款號、品牌、季節、描述）
- 版本列表（Revision History）
- 關聯的 Sample Request
- BOM、Spec、報價連結

### 5.7 文件處理流程

```
上傳 → AI 分類 → 確認分類 → AI 提取 → 審校內容 → 完成
```

| 操作 | 說明 |
|------|------|

| 點擊「驗證 BOM」| 檢查完整性並標記已驗證 |

| 階段 | 欄位 | 說明 |
|------|------|------|
| 1. 初始 | `initial_consumption` | AI 提取或手動輸入 |
| 2. 確認 | `confirmed_consumption` | 打樣前確認 |
| 3. 實際 | `actual_consumption` | 打樣後實測 |
| 4. 大貨 | `bulk_consumption` | 大貨生產用量 |

### 6.5 頂部導航

- **📦 BOM 物料**：當前頁面
- **📏 Spec 尺寸**：跳轉尺寸規格頁

| `add task [內容]` | 新增任務 | `add task 催促 LM7B24S 面料` |
| `add note [內容]` | 新增筆記 | `add note 客戶要求修改尺寸` |
| `draft email [款號]` | 生成 PO 郵件草稿 | `draft email LW1FLWS` |

### 17.4 操作方式

1. 點擊頁面右下角的 ✨ 按鈕開啟小助理
2. 在輸入框輸入指令
3. 按 Enter 或點擊「送出」執行
4. 查看回應結果
5. 點擊 ✕ 關閉對話框

### 17.5 使用場景

- **每日檢查**：`overdue` + `this week` 快速了解待處理項目
- **追蹤進度**：`check LW1FLWS` 查詢特定款式狀態
- **任務管理**：`add task` 記錄待辦事項
- **郵件撰寫**：`draft email` 快速生成採購單催促郵件

## 附錄 B：常見問題

### Q1: 上傳文件失敗？
**A:** 檢查文件格式（僅支援 PDF）和大小（最大 50MB）。

### Q2: AI 提取結果不準確？
**A:** 在審校頁面手動修正，系統會持續學習改進。

### Q3: 看板卡片無法推進？
**A:** 確認前置步驟已完成（如 BOM 已驗證才能生成 MWO）。

### Q4: 匯出的 PDF 中文亂碼？
**A:** 確認系統已安裝微軟雅黑字體（msyh.ttc）。

### Q5: Email 發送失敗？
**A:** 聯繫管理員檢查 Email 設定（SMTP 配置）。
