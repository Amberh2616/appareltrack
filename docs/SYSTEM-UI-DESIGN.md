## 🎯 核心設計理念

### 這是一個**管理系統**，不是聊天機器人

- **主要功能**：專案管理、Tech Pack 管理、BOM 管理、Sample 追蹤、採購管理
- **AI 角色**：背景智能助手（自動解析、智能填充、風險預警）
- **用戶操作**：表格編輯、表單填寫、看板拖拽、文件上傳
- **介面風格**：借鑒 Claude 的側邊欄 + Notion 的工作區 + Linear 的現代感

## 🤖 AI 功能整合方式

### AI 不是對話，而是**背景助手**

#### 1. Tech Pack 上傳 → 自動解析

```
用戶操作:
[📤 上傳 Tech Pack] → 選擇 PDF → 上傳

系統流程:
上傳 → 顯示進度條 → AI 解析（背景） → 解析完成通知

結果展示:
Tech Pack 列表中顯示 "✅ 已解析"
點擊查看 → 顯示解析結果表格（可編輯）

無需對話！直接看結果！
```

#### 2. BOM 編輯 → AI 智能建議

```
用戶操作:
在 BOM 表格中編輯數據

AI 功能:
背景監控編輯 → 發現異常/優化機會 → 彈出建議框

建議框:
┌────────────────────────────────────┐
│ 💡 AI 建議                         │
│ 主料損耗率建議調整為 7%            │
│ 理由: 歷史數據顯示針織布料...      │
│                                    │
│ [接受建議] [忽略] [了解更多]      │
└────────────────────────────────────┘
```

#### 3. Sample Fit Comment → AI 自動摘要

```
用戶操作:
上傳 Sample 照片 + 輸入 Fit Comments

AI 功能:
自動分析照片 + 文字 → 生成摘要 → 自動填入表格

顯示方式:
┌────────────────────────────────────┐
│ 🤖 AI 分析摘要                     │
│ 主要問題: 肩寬、腰圍               │
│ 嚴重程度: 中等                     │
│ 預計修改天數: 3                    │
│                                    │
│ [查看詳細分析]                     │
└────────────────────────────────────┘
```

#### 4. 採購建議 → AI 主動推送

```
系統流程:
AI 監控 BOM + 庫存 + 交期 → 發現需採購 → 推送到通知

通知展示:
🔔 (3)
┌────────────────────────────────────┐
│ 🤖 採購建議                        │
│ LW1FLWS 需要採購主料               │
│ 建議數量: 2800 yards               │

│                                    │

```

## 📝 總結

### 關鍵差異：管理系統 vs 對話系統

| 特性 | 對話系統 ❌ | 管理系統 ✅ |
|------|-------------|-------------|
| 主介面 | 聊天氣泡 | 表格/表單/看板 |
| 用戶操作 | 輸入文字 | 點擊/編輯/拖拽 |
| AI 角色 | 對話夥伴 | 背景助手 |
| 數據展示 | 文字回應 | 結構化表格 |
| 功能觸發 | 提問 | 按鈕/操作 |

### 功能串接簡單因為：
✅ 表格直接對應資料庫（CRUD）
✅ AI 結果直接填入欄位（無需對話）
✅ 操作流程清晰（上傳 → 解析 → 審核 → 確認）
✅ 狀態管理簡單（表單狀態）
